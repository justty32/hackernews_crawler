# AI Agent Operating Guide for Hacker News Crawler

This document provides technical specifications and operational workflows for AI Agents to interact with the Hacker News Crawler tool.

---

## 1. Tool Overview
The Hacker News Crawler is a modular system for intelligence gathering. It fetches Hacker News stories, generates LLM summaries, filters them via a 3-layer expert system, and sends notifications.

## 2. Command Line Interface (CLI)
All operations are routed through `main.py`.

| Command | Action | Persistence | Primary Output |
| :--- | :--- | :--- | :--- |
| `python main.py crawl` | Fetches top stories and comments. | Single Task | `data/raw/{story_id}.txt` |
| `python main.py summarize` | Generates summaries for raw data. | Single Task | `data/summary/{YYYY-MM}/summary_{story_id}.txt` |
| `python main.py monitor` | Watches for new summaries and evaluates. | Persistent | `data/notifications/notification_{timestamp}.txt` |
| `python main.py organize` | Compresses and archives old data. | Single Task | `data/archive/{category}/{id}.zip` |
| `python main.py all` | Runs `crawl` then `summarize`. | Single Task | Both raw and summary files. |

## 3. Data Schema & Specifications

### 3.1 Raw Data (`data/raw/*.txt`)
Used as the source of truth for comments.
- **Format**: Plain text with headers.
- **Header Keys**: `Title:`, `URL:`, `Score:`, `Comments Count:`.
- **Comment Marker**: `\n- {author}: {text}`.

### 3.2 Summary Data (`data/summary/YYYY-MM/*.txt`)
Intermediate intelligence generated by LLM.
- **File Naming**: `summary_{story_id}.txt`.
- **Header Keys**: `Original Story:`, `Story ID:`, `Generated at:`.
- **Content**: Multi-point analysis in Traditional Chinese.

### 3.3 Notifications (`data/notifications/*.txt`)
**The primary output for Agent consumption.** These represent items that passed all 3 filter layers (Keyword, Threshold, AI Expert).
- **Format**:
  ```text
  Subject: [HN Match] {Title}
  ==============================
  🚀 感興趣文章提醒: {Title}
  🔗 HN 連結: {URL}
  💬 留言數量: {Count}
  💡 AI 評估原因: {Detailed Reason}
  ------------------------------
  📄 文章總結:
  {Full Summary Content}
  ```

## 4. Operational Workflow for Agents

### 4.1 Intelligence Gathering Cycle
1.  **Configuration**: Modify `config.yaml` to set `monitoring.rules.expert_check_prompt`.
2.  **Batch Processing**: Execute `python main.py all`.
3.  **Active Monitoring**: Run `python main.py monitor` in the background (detached process recommended).
4.  **Consumption**: Poll the `data/notifications/` directory for new intelligence files.

### 4.2 Maintenance
- Periodically run `python main.py organize` to prevent disk bloat.
- Monitor `data/archive/` for historical research.

## 5. Error Handling for Agents
- **Exit Codes**: Non-zero exit codes indicate failure. Check stdout for `Error:` or `Failed:` prefixes.
- **Encoding**: All files are written in `UTF-8` (with SIG support for config). Ensure your environment handles UTF-8 correctly.
- **LLM Rate Limits**: `summarizer.py` and `notifier.py` include `time.sleep(1)` to mitigate API rate limiting.

## 6. Configuration Requirements
- `.env` must contain valid `OPENAI_API_KEY` (or compatible API key).
- `config.yaml` must be properly initialized. (Refer to `config.yaml.example` if available).
